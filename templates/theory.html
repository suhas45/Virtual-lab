ER DIAGRAM

An Entity may be an object with a physical existence – a particular person, car, house, or employee – or it may be an object with a conceptual existence – a company, a job, or a university course.

Strong Entity: A Strong Entity is a type of entity that has a key Attribute. Strong Entity does not depend on other Entity in the Schema. It has a primary key, that helps in identifying it uniquely, and it is represented by a rectangle.

Weak Entity: An Entity type has a key attribute that uniquely identifies each entity in the entity set. But some entity type exists for which key attributes can’t be defined. These are called Weak Entity types.

This ER diagram for bus reservation represents the model of the bus ticket booking system entity. The entity-relationship diagram of the bus ticket booking system shows all the visual instruments of database tables and the relations between Booking, bus route, bus, set, etc. ER diagram for bus reservation used structure data to define the relationships between structured data groups of bus ticketing booking system functionalities. The main entities of the bus ticket booking system are bus, Booking, customer, bus route, ticket booking, and seat. This diagram implements indexing on each bus ticket booking system table for fast query execution.
